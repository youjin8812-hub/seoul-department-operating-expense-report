# 산출물 형식

## HTML 대시보드 (`output/dashboard.html`, GitHub Pages: `docs/index.html`)

Plotly.js를 인라인 포함한 단일 파일로, 인터넷 연결 없이 브라우저에서 바로 열립니다.

- **KPI 카드 5개**: 총 집행액, 집행 건수, 부서 수, 최고 집행월, 동일기간 비교가능 부서 수
- **필터 6종**: 월 다중선택, 비목 다중선택, 부서 검색·선택, 동일기간 보유 부서만 보기, 상위 N, 지표 전환(집행액/건수/평균금액)
- **차트 8종**: 월별 현황, 월별 비목 구성, 부서 비교, 부서×비목 히트맵, 부서 규모-활동성 산점도, 집중도 파레토, 금액 분포, 반복 패턴
- **자동 해석문**: 각 차트 아래 데이터 기반 사실 1~2문장(필터 변경 시 재계산)
- **실시간 자체 검증 배지**: 페이지 로드 시 필터 미적용 총액이 원본과 일치하는지 브라우저에서 즉시 확인
- **개인정보**: 임베드 데이터에 해당월·비목·전체부서명·집행금액 4개 필드만 포함 — 전화번호·작성자·집행대상은 데이터 자체에 없음

## DOCX 상세 보고서 (`output/*.docx`)

A4 세로, 여백 15mm, 본문 맑은 고딕 9.5~10pt·표 8~9pt. 표는 열 너비를 명시적으로 지정하고 행 분할 방지·헤더 반복을 적용해 페이지 경계에서 잘리지 않도록 구성했습니다.

| 페이지 | 내용 |
|---|---|
| 1 | 표지, 분석 개요, KPI, 핵심 결과 요약 |
| 2 | 월별 집행 현황(누적 막대) |
| 3 | 월별 비목 구성 변화(100% 누적 막대), 비목별 집행 현황 표 |
| 4 | 동일기간 부서 비교표(전체), 비교 조건과 해석 |
| 5 | 부서×비목 구성 히트맵 |
| 6 | 부서별 집행액·건수 산점도 |
| 7 | 집중도 파레토 |
| 8 | 금액 분포 및 고액 집행 후보 |
| 9 | 반복 집행 패턴 |
| 10 | 핵심 시사점, 검토 사항(데이터 품질·분석 한계) |

## HWPX 보고서 (`output/*.hwpx`)

DOCX를 한컴오피스 한글의 COM 자동화(`HWPFrame.HwpObject`)로 열어 `SaveAs(..., "HWPX")`로 변환합니다. **Windows + 한컴오피스 한글이 설치된 환경에서만 생성되는 선택적 산출물**이며, 조건 미충족 시 자동으로 건너뛰고 HTML·DOCX는 정상 생성 상태를 유지합니다.

- 변환 후 새 COM 세션으로 다시 열어(`Open`) 정상 로딩을 확인합니다.
- DOCX 내 이미지(`word/media/`)와 HWPX 내 이미지(`BinData/`)를 SHA256 해시로 대조해 무손실 보존 여부를 확인합니다.
- 페이지 수는 한컴 엔진 기준으로 DOCX·HWPX 양쪽에서 실측해 비교합니다(다르다고 오류로 단정하지 않음 — 렌더링 엔진 특성상 정상적으로 차이 날 수 있음).

## 검증 보고서 (`output/*_report.md`, 공개 요약: `docs/validation_summary.md`)

| 보고서 | 검증 내용 |
|---|---|
| 데이터·개인정보 검증 | 원본 CSV 재계산 값 ↔ 분석 JSON ↔ DOCX 수치 일치, 동일기간 비교 조건 준수, 개인정보 원문 비노출 |
| 레이아웃 검증 | DOCX 여백·표 너비·행 분할 방지·헤더 반복·이미지 크기 등 XML 구조 정적 검증 |
| HWPX 변환 검증 | 재오픈 성공 여부, 핵심 수치 DOCX-HWPX 일치, 이미지 보존, 폰트 보존, 개인정보 비노출 |

공개 저장소에는 원시 로그(`*_log.txt`)와 전체 보고서 대신, `docs/validation_summary.md`에 핵심 판정 결과만 요약해 포함합니다.
